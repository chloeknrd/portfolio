Masterthesis: Fridays for Future und YouTube

Innerhalb meiner Masterthesis habe ich die Bewegung „Fridays for Future“ (FFF) auf der Plattform YouTube im Laufe der Zeit und im Vergleich zu anderen Protestbewegungen analysiert. Hierzu habe ich zuerst via der YouTube-API auf Basis einer Query eine Liste von den relevanten YouTube Videos per ID inklusive ersten Metadaten gesammelt.
Auf Basis der Liste an IDs habe ich die Transkripte der Videos gesammelt, aufbereitet und in einer Datenbank abgespeichert. Zusätzlich habe ich die Kategorie der jeweiligen Kanäle gesammelt, um sowohl die Themenverteilung über Zeit als auch über Kanalkategorien analysieren zu können.
Innerhalb der Analyse habe ich BERTopic basiertes Topic Modelling betrieben: Zuerst habe ich die Topics selbst generiert, um dann die Nähe der jeweiligen Topics zu bestimmen, um Clusters zu identifizieren. Diese Cluster habe ich dann bzgl. Veränderung der Verteilung über Zeit und Kanalkategorien analysiert.
Die Ergebnisse zeigen, wie die Bewegung die Themenbreite an Videos über Umwelt und Klimawandel reduzierte und die Aufmerksamkeit sich auf die Bewegung selbst verschoben hat. Dennoch zeigt die große Bandbreite an Topics vor FFF, dass ein Grundinteresse und eine geteilte Wissensbasis bestand und somit die Grundlage für die Forderungen der Bewegung.

Data Scraping und Management

YouTube API

Der Data Scraping Process ist aufgeteilt in drei Schritte:
- Sammeln der relevanten IDs an Videos und YouTube-Kanälen, zusammen mit ersten Metadaten von Videos.
- Sammeln an Transkripten via der IDs von YouTube-Videos
- Sammeln der Kategorien der jeweiligen YouTube-Kanälen

Video- und Kanal-IDs

YouTubes API hat im Vergleich zu anderen APIs wie von der New York Times oder vom Guardian eine andere Architektur und Aufbau. Der größte Unterschied, der auch einen anderen Ansatz notwendig macht, ist der unterschiedliche Ansatz bzgl. Pagination. Während die anderen besagten APIs maximale Seitanzahlen verwenden, um auf weitere Seiten an Suchergebnissen zu verweisen, verwendet die YouTube-API Tokens, die auf die darauf folgende Seite verweisen. Dadurch muss auch die Automatisierung anders aufgestellt werden, da eine Iteration über die Seitenzahlen nicht möglich ist.
Durch die Limitation eines Zeitraums ist es möglich, eine Endlosschleife aufzusetzen, bis der Endpunkt erreicht worden ist. Die Ergebnisse der Query werden zuerst in einem Dictionary gespeichert und dann umformatiert in einen gesonderten Pandas DataFrame. Besagter DataFrame wird an den großen DataFrame angehangen, in welchem alle Scraping-Ergebnisse dieser automatisierten Query gespeichert sind.

Code Beispiel 

while True:
    if next_token:
        if date_begin >= date_split:

        request = youtube.search().list(

						items(id(videoId),

    else:

            part="id, snippet",

            relevanceLanguage = 'de',

						items(id(videoId),

        ).execute()    

    for i in range(len(request['items'])):

        vid_id = request['items'][i]['id']['videoId']
        Publish_Date = request['items'][i]['snippet']['publishedAt']
        Channel_ID = request['items'][i]['snippet']['channelId']
        Title = request['items'][i]['snippet']['title']

    else:

    if date_begin > date_finish:

YouTube Transkripte

Die Transkripte stellen den elementaren Teil der Daten dar, und mit der Liste an IDs der Videos ist der nächste Scraping-Ansatz simpler. Dennoch bleiben weitere Probleme: Manche Videos besitzen kein Transkript, weswegen sie nciht nur für die Analyse ungeeignet sind, sondern auch bei der Verwendung der Batch-Scraping Funktion einen Error auslösen.
Somit muss eine weitere Schleife aufgestellt werden, die die Funktion verwendet, nur für ein Video die Transkripte zu sammeln. Mit einem "try-except"-Ansatz werden Videos ohne Transkripte mit "NaN" markiert und in beiden Fällen mit der ID in einem Tupel an eine Liste angehangen und später auch in ein DataFrame konvertiert.
Zuletzt fehlt noch die Bereinigung der Daten, da die Ergebnisse mit zusätzlicher Formattierung kommen, z.B. Start- und Endzeitpunkt des jeweligen Satzes. Da diese Informationen in einem Dictionary mit Keys gespeichert sind, werden diese mit einer Schleife erneut gefiltert und mit der Video-ID als Tupel in einer Liste , und dann in einem DataFrame gespeichert.

Code Beispiel 

video_list = yt_data['vid_id'].to_list()
list_capt = []

for i in range(len(video_list)):
    try:
        req_cap = YouTubeTranscriptApi.get_transcript(video_id=video_list[i], languages=['de'])
    except:
        req_cap = float("nan")
    req_tuple = (video_list[i], req_cap)
    list_capt.append(req_tuple)
    print(i)
cap_df = pd.DataFrame(list_capt, columns=['Vid_Id', 'Trans'])

cap_df_clean = cap_df.dropna()
cap_df_clean = cap_df_clean.reset_index(drop=True)
k = 'text'
trans_list = []
for i in range(len(cap_df_clean.loc[:,'Trans'])):
    res = [d.get(k) for d in cap_df_clean.loc[i,'Trans'] if k in d]
    res = ' '.join(res)
    res_tup = (cap_df_clean.loc[i, 'Vid_Id'], res)
    trans_list.append(res_tup)
script_df = pd.DataFrame(trans_list, columns=['Vid_Id', 'Trans'])
